# 📊 DePaul University Admissions Data Analysis & Dashboard Project

## 📌 Project Overview

This project was completed as part of the **Excelerate Early Internship Program (Data Visualization Track)**. It focuses on analyzing a real-world admissions dataset from DePaul University and transforming raw, unstructured data into meaningful insights through data cleaning, exploratory analysis, and interactive dashboard development.

The project demonstrates a complete **end-to-end data analytics workflow**, from raw data exploration to insight-driven dashboard reporting.

---

## 🎯 Problem Statement

The dataset provided was large, complex, and not analysis-ready. Key challenges included:

- Over **7,000+ rows and 80+ columns**
- High volume of **missing values (null entries)**
- **Duplicate records**
- Inconsistent categorical data (e.g., country, region, degree)
- Improperly formatted fields, especially date columns
- Redundant and low-value variables

These issues made accurate analysis and reporting difficult without proper data preparation.

---

## 🎯 Aim of the Analysis

The objectives of this project were to:

- Clean and prepare the dataset for analysis  
- Explore the dataset to uncover trends and patterns  
- Define and calculate key performance indicators (KPIs)  
- Build an interactive dashboard for decision-making  
- Translate findings into clear and actionable insights  

---

## 🛠️ Tools & Technologies Used

- **Microsoft Excel** – Data exploration and cleaning  
- **SQL** – Data querying and transformation  
- **Power BI** – Dashboard development and visualization  
- **GitHub** – Project documentation and version control  

---

## 🔍 Project Workflow

### 📅 Week 1 – Data Exploration & Understanding
- Reviewed dataset structure and column types  
- Identified missing values, duplicates, and inconsistencies  
- Created a structured **data dictionary**  
- Performed initial exploratory data analysis  

---

### 🧹 Week 2 – Data Cleaning & Preparation
- Removed irrelevant and redundant columns  
- Handled missing values and null records  
- Standardized categorical fields (country, region, degree)  
- Converted date columns into proper formats  
- Prepared a **clean, analysis-ready dataset**  

---

### 📊 Week 3 – Dashboard Development
- Built an interactive **Power BI dashboard**  
- Created key performance indicators (KPIs):
  - Total Applications  
  - Total Admitted  
  - Admission Rate  
- Designed visuals for:
  - Country and region distribution  
  - Degree and program preferences  
  - Intake trends  
  - Admission outcomes  
- Added interactivity:
  - Filters and slicers  
  - Drill-down functionality  
  - Tooltips and navigation buttons  

---

## 📈 Key Insights

- Applications are **concentrated in a few countries and regions**, showing dependence on specific markets  
- **Bachelor’s degree programs dominate** application volume  
- Certain colleges and first-choice programs receive significantly higher interest  
- Admission rate (~65%) indicates **moderate selectivity**  
- Outreach and engagement tracking varies across counselors  

---

## 💡 Recommendations

- Expand recruitment efforts in **high-performing regions** while targeting underrepresented markets  
- Focus on **high-demand programs** to improve conversion  
- Improve outreach tracking to better evaluate **counselor performance**  
- Standardize admission indicators for better analysis  
- Improve data collection to reduce missing values
